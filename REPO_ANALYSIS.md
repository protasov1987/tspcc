# Обзор репозитория

Этот проект представляет собой локальное веб‑приложение для управления маршрутными технологическими картами (ТСЗП), включающее браузерный интерфейс и минималистичный сервер на Node.js без внешних зависимостей.

## Ключевые компоненты
- **Статический фронтенд**: `index.html` определяет навигацию по вкладкам («Дашборд», «Тех. карты», «Трекер», «Архив»), формы поиска/фильтрации и панели редактирования справочников операций и участков производства. Вёрстка использует подключённый файл стилей `style.css`.【F:index.html†L1-L81】
- **Логика клиента**: `app.js` реализует работу с состоянием карт, валидацию вложений, генерацию кодов операций, отслеживание времени выполнения и обмен с серверным API (`/api/data`). Код отвечает за рендеринг таблиц, модальных окон, историю изменений и работу со вложениями (до 15 МБ, типы pdf/doc/docx/jpg/png/zip/rar/7z).【F:app.js†L1-L120】
- **Серверное API**: `server.js` поднимает HTTP‑сервер (по умолчанию на `localhost:8000`), обслуживает статические файлы и JSON‑эндпоинты, управляет файловым хранилищем данных `data/database.json` и проверяет вложения по размеру/расширению. При отсутствии базы создаёт демо‑данные с маршрутной картой, справочником операций и производственных участков.【F:server.js†L1-L112】
- **Файловая база**: `db.js` предоставляет примитивный движок с очередью записи и нормализацией данных, на котором основаны чтение/запись `database.json` и сохранение вложений.

## Запуск и данные
- Приложение стартует командой `npm start`, использует только встроенные модули Node.js и хранит всё состояние в `data/database.json`, что позволяет обслуживать несколько клиентов из разных браузеров через один сервер.【F:README.md†L1-L23】【F:package.json†L1-L10】
- При первом запуске автоматически создаются демо‑данные; для переноса состояния на другой экземпляр достаточно скопировать файл базы из каталога `data/`.【F:README.md†L17-L27】

## Особенности функциональности
- Поддерживаются вложения к картам (ограничения 15 МБ и набор расширений совпадают на клиенте и сервере).【F:app.js†L15-L21】【F:server.js†L10-L13】
- Доступен поиск и фильтрация карт по номеру заказа/штрихкоду и статусам, управление статусами операций (старт/пауза/продолжение/завершение) и перенос выполненных карт в архив с возможностью повторного запуска копией EAN‑13.【F:index.html†L23-L74】
