# Realtime Directories / Security Checklist

Короткий regression-smoke checklist для третьей фазы после изменений в:
- `directories`
- `users`
- `accessLevels`
- security live и current-user permissions sync

## Operations

1. `/operations` + `/operations`
   - создать операцию -> появляется без `F5`
   - изменить операцию -> обновляется без `F5`
   - удалить операцию -> исчезает без `F5`

2. Dependency refresh
   - route selectors видят новые и изменённые операции без reload

## Areas

1. `/areas` + `/areas`
   - create/update/delete без `F5`

2. Если во второй вкладке открыта `/operations`
   - labels и selects по участкам обновляются без reload

## Departments

1. `/departments` + `/departments`
   - create/update/delete без `F5`

2. Если во второй вкладке открыта `/employees`
   - таблица сотрудников отражает изменения подразделений без reload

3. Если открыта `/departments`
   - счётчики сотрудников остаются корректными

## Shift Times

1. `/shift-times` + `/shift-times`
   - изменение времени смены видно без `F5`
   - изменение времени обеда видно без `F5`

2. Если во второй вкладке открыта production-страница с переключателями смен
   - подписи времени смен обновляются без reload

## Employees

1. `/employees` + `/employees`
   - смена подразделения пользователя видна без `F5`

2. Если во второй вкладке открыта `/departments`
   - счётчики сотрудников обновляются без `F5`

3. Если во второй вкладке открыта `/users`
   - изменения по пользователю доходят без `F5`

## Users

1. `/users` + `/users`
   - создать пользователя -> появляется без `F5`
   - изменить пользователя -> обновляется без `F5`
   - удалить пользователя -> исчезает без `F5`

2. Derived security state
   - колонка `Уровень` синхронизируется без `F5`
   - после `F5` состояние совпадает с live-состоянием

## Access Levels

1. `/accessLevels` + `/accessLevels`
   - создать уровень доступа -> появляется без `F5`
   - изменить уровень доступа -> обновляется без `F5`

2. Если во второй вкладке открыта `/users`
   - названия уровней у пользователей обновляются без `F5`

## Current User Permissions Sync

1. Изменить права другого пользователя
   - текущая вкладка не должна внезапно редиректить

2. Изменить права текущего пользователя, оставив текущий route разрешённым
   - меню и readonly обновляются
   - route не меняется

3. Изменить права текущего пользователя так, чтобы текущий route стал запрещённым
   - происходит мягкий redirect на home route
   - без `F5`
   - без редиректа на dashboard по умолчанию

4. Изменить `landingTab` текущего пользователя
   - текущий route не должен меняться, если он разрешён
   - home route должен стать новым для последующих redirect/fallback
