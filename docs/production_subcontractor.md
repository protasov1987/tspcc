# Участок `Субподрядчик`

Документ фиксирует специальную логику для операций, запланированных на участок с типом `Субподрядчик`.

## Область действия

- Спецлогика включается только если `area.type === 'Субподрядчик'`.
- Для остальных участков поведение не меняется.

## Планирование

- Ручное и автоматическое планирование строят непрерывную цепочку по сменам без разрывов.
- Цепочка может переходить через границы суток.
- В стартовой смене используется остаток времени смены, дальше заполняются следующие смены.
- При ручном планировании перегрузка участка допускается.
- При автопланировании старт в `CLOSED` и `LOCKED/FIXED` запрещён, загрузка выше 100% не допускается.
- После старта цепочка может проходить сквозь `CLOSED`, `LOCKED` и `FIXED` смены. Время этих смен входит в длительность.
- Если расчётный конец цепочки попадает в недоступную смену, конец переносится на следующую доступную.

## Цепочки

- Для каждой цепочки используется отдельный `subcontractChainId`.
- Одна `routeOpId` может иметь несколько цепочек, если у них разный состав изделий/ОС/ОК.
- Если планируется не весь объём, пользователь обязан выбрать конкретные изделия/ОС/ОК в модалке.
- Пересечение составов изделий между цепочками запрещено.

## Отображение

- На плане цепочка рисуется как сквозной бар через соседние ячейки.
- Полная карточка и кнопка удаления показываются только в первой и последней смене цепочки.
- В промежуточных сменах показывается только бар без текста и без кнопки удаления.
- Продлённые через закрытие смены фрагменты окрашиваются светло-красным полупрозрачным цветом.

## Ограничения

- Drag-and-drop для цепочек субподрядчика запрещён.
- Любой ручной перенос между сменами, датами и участками запрещён.
- Удаление цепочки доступно только до старта операции и только из первой или последней смены.

## Выполнение

- На участках `Субподрядчик` исполнитель может быть не назначен.
- Действия `Начать`, `Пауза`, `Продолжить`, `Завершить` доступны:
  - мастеру смены соответствующей даты и смены;
  - назначенному исполнителю операции.
- Фактическое время считается как рабочее время между `Начать` и `Завершить`.
- Время на `Пауза` в факт не входит.

## Закрытие смены

- Для `IN_PROGRESS` и `PAUSED` строк субподрядчика закрытие смены разрешено только если у цепочки уже есть следующий фрагмент.
- Если следующего фрагмента нет, в строке доступен только чекбокс `Перенести`.
- Продление ищет следующую доступную смену, пропуская закрытые и зафиксированные.
- Все строки субподрядчика в close-page и snapshot подсвечиваются синим.

## Завершение

- Завершение цепочки возможно только когда все изделия/ОС/ОК этой цепочки имеют статус `GOOD`, `DELAYED` или `DEFECT`.
- После завершения будущие фрагменты цепочки удаляются.
- Конец цепочки переносится на фактическую смену завершения.

## Логи

- В shift log и в log карты пишутся отдельные события:
  - создание цепочки;
  - удаление цепочки;
  - продление цепочки;
  - завершение цепочки;
  - удаление будущих фрагментов после завершения.
